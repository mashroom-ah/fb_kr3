# Что студент должен добавить самостоятельно

Этот файл нужен для того, чтобы вы не просто запустили готовый шаблон, а доработали его и показали понимание логики PWA-приложения.

## 1. Доработка логики TODO-списка

### 1.1. Редактирование задачи

Что нужно сделать:

- добавить кнопку **«Редактировать»** у каждой задачи;
- реализовать изменение текста существующей задачи;
- после редактирования сохранить обновлённый массив в `localStorage`.

Где менять:

- `app.js` → функция `createTaskElement()`;
- `app.js` → обработчик `taskList.addEventListener('click', ...)`;
- при необходимости добавить отдельную функцию `editTask(taskId, newText)`.

### 1.2. Фильтрация списка

Что нужно сделать:

- добавить режимы отображения:
  - все задачи;
  - только активные;
  - только выполненные.

Где менять:

- `index.html` → добавить кнопки фильтров;
- `app.js` → изменить `renderTasks()` так, чтобы она учитывала выбранный фильтр.

### 1.3. Подтверждение удаления

Что нужно сделать:

- перед удалением одной задачи или группы выполненных задач добавить подтверждение действия.

Где менять:

- `app.js` → `deleteTask(taskId)`;
- `app.js` → обработчик удаления выполненных задач.

## 3. Доработка Service Worker

### 3.1. Улучшить стратегию кэширования

Сейчас в шаблоне используется упрощённая стратегия `Cache First`.

Что нужно сделать:

- определить, какие ресурсы кэшируются заранее, а какие динамически;
- реализовать отдельный runtime-cache для части GET-ресурсов;
- не кэшировать подряд всё без разбора.

Где менять:

- `sw.js` → обработчик `fetch`.

### 3.2. Улучшить fallback для офлайн-режима

Что нужно сделать:

- вместо простого текстового ответа вернуть более удобный fallback;
- при желании создать отдельную офлайн-страницу.

Где менять:

- `sw.js` → блок `.catch(...)` внутри `fetch`.

### 3.3. Обновление версии кэша

Что нужно сделать:

- изменить имя кэша;
- проверить, как приложение удаляет старые версии;
- объяснить, зачем это делается.

Где менять:

- `sw.js` → `CACHE_NAME`;
- `sw.js` → обработчик `activate`.

## 5. Что вы должны уметь объяснить по итогам работы

После выполнения практики вы должны уметь объяснить:

1. зачем нужен `manifest.json`;
2. зачем нужен `Service Worker`;
3. чем отличается `localStorage` от `Cache Storage`;
4. почему проект нужно запускать через локальный сервер, а не через `file://`;
5. что происходит на этапах `install`, `activate`, `fetch`;
6. почему после изменения `sw.js` иногда нужно обновлять кэш;
7. почему установка PWA в Chromium и Safari выглядит по-разному.

---

Если хотите усилить работу, дополнительно можно реализовать:

- тёмную / светлую тему;
- отдельный экран или модальное окно с инструкцией по установке.
