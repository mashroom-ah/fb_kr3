# Учебный репозиторий для практик 13–14: PWA, Service Worker, Manifest, offline-first

## Назначение репозитория

Этот репозиторий представляет собой учебную заготовку для практических занятий по теме Progressive Web App.
В основе лежит простое приложение-список задач, на котором удобно разбирать:

- работу `localStorage`;
- регистрацию и жизненный цикл `Service Worker`;
- кэширование статических ресурсов;
- настройку `manifest.json`;
- установку сайта как PWA-приложения;
- поведение приложения при отключении сети.

Шаблон сделан **не полностью завершённым**: основная база уже работает, а часть функций оставлена студентам как самостоятельная доработка.

---

## Что уже реализовано

В текущей версии шаблона уже есть:

1. статичный интерфейс с хедером, основным содержимым и футером;
2. базовый TODO-менеджер;
3. сохранение задач в `localStorage`;
4. добавление, удаление и отметка задачи как выполненной;
5. удаление всех выполненных задач;
6. подсчёт статистики по задачам;
7. отображение онлайн/офлайн статуса;
8. регистрация `Service Worker`;
9. предварительное кэширование основных статических ресурсов;
10. подключённый `manifest.json`;
11. набор локальных иконок;
12. кнопка установки PWA для Chromium-браузеров через `beforeinstallprompt`;
13. текстовая подсказка по установке для Safari;
14. случайные мотивационные цитаты в футере.

---

## Что важно с точки зрения методички

По логике практики студент должен **не только подключить `manifest.json` и `Service Worker`, но и проверить реальную установку PWA**.

В материалах практики отдельно фигурирует проверка установки:
- найти кнопку «Установить» или встроенный значок установки в браузере;
- установить приложение;
- убедиться, что после установки оно открывается в отдельном окне.

Поэтому в обновлённой версии шаблона:
- добавлена собственная кнопка установки PWA для Chromium-браузеров;
- оставлена подсказка для Safari, потому что там установка выполняется через меню браузера, а не через событие `beforeinstallprompt`.

## Что студент должен понять по этой работе

### 1. Работа с `localStorage`
Студент должен понимать:
- почему данные списка задач не исчезают после перезагрузки страницы;
- почему массив объектов нужно сохранять через `JSON.stringify()`;
- почему при чтении используется `JSON.parse()`;
- чем `localStorage` отличается от `Cache Storage`.

### 2. Работа `Service Worker`
Студент должен понимать:
- что `Service Worker` — это отдельный скрипт, работающий в фоне;
- что делают этапы `install`, `activate`, `fetch`;
- как происходит предварительное кэширование;
- почему после изменения `sw.js` иногда приходится обновлять кэш и перезапускать приложение.

### 3. Назначение `manifest.json`
Студент должен понимать:
- как браузер получает имя приложения, цвета и иконки;
- зачем нужны поля `name`, `short_name`, `display`, `start_url`, `icons`;
- почему манифеста самого по себе недостаточно без `Service Worker`.

### 4. Установка PWA
Студент должен понимать:
- что установка может выглядеть по-разному в разных браузерах;
- что в Chromium поддерживается событие `beforeinstallprompt`;
- что в Safari установка обычно выполняется через **File → Add to Dock**;
- как проверить, что приложение после установки открывается в отдельном окне.

## Как запустить проект

### Вариант 1. Через VS Code Live Server
1. Откройте папку проекта в VS Code.
2. Убедитесь, что установлено расширение **Live Server**.
3. Запустите `index.html` через **Open with Live Server**.

### Вариант 2. Через Python
В корне проекта выполните:

```bash
python -m http.server 5500
```

После этого откройте:

```text
http://localhost:5500
```

### Важно
Не запускайте проект двойным кликом по `index.html` через `file://`.
В таком режиме `Service Worker` обычно не работает.

## Как проверять работу практики

### 1. Проверка Service Worker
1. Откройте приложение в браузере.
2. Нажмите `F12`.
3. Перейдите в `Application` → `Service Workers`.
4. Убедитесь, что Service Worker зарегистрирован и активен.

### 2. Проверка кэша
1. В DevTools откройте `Application` → `Cache Storage`.
2. Убедитесь, что в кэше появились HTML, CSS, JS, manifest, иконки и картинка хедера.

### 3. Проверка офлайн-режима
1. В DevTools переведите сеть в offline.
2. Перезагрузите страницу.
3. Убедитесь, что интерфейс по-прежнему открывается.
4. Проверьте, что список задач отображается и состояние хранится в `localStorage`.

### 4. Проверка Manifest
1. Откройте `Application` → `Manifest`.
2. Проверьте название, иконки, `start_url`, `display`, `theme_color`.

### 5. Проверка установки PWA
#### Chrome / Edge / другие Chromium-браузеры
1. Откройте приложение.
2. Найдите кнопку установки в адресной строке или нажмите кнопку **«Установить PWA»** внутри интерфейса.
3. Установите приложение.
4. Убедитесь, что оно открывается как отдельное окно.

#### Safari
1. Откройте сайт в Safari.
2. Используйте **File → Add to Dock**.
3. Запустите созданное приложение.
4. Проверьте, что оно открывается как отдельное окно.

## На что обратить внимание в коде

### `index.html`
Здесь важно проследить:
- где подключается `manifest.json`;
- где подключаются локальные иконки;
- как устроены статические элементы интерфейса;
- где расположена кнопка установки PWA.

### `app.js`
Здесь важно проследить:
- как читаются и сохраняются задачи;
- как происходит полная перерисовка списка;
- как работает делегирование событий;
- как обрабатывается `beforeinstallprompt`;
- где регистрируется `Service Worker`.

### `sw.js`
Здесь важно проследить:
- какие ресурсы попадают в pre-cache;
- как удаляются старые версии кэша;
- как работает стратегия `Cache First`;
- как реализован офлайн-fallback.

### `manifest.json`
Здесь важно проследить:
- какие поля влияют на установку PWA;
- как задаются пути к иконкам;
- как `display: standalone` влияет на режим запуска.

## Рекомендуемая последовательность изучения

1. Сначала запустить проект и убедиться, что он работает.
2. Изучить `index.html` и `styles.css`.
3. Разобрать `app.js` и понять работу `localStorage`.
4. Проверить данные в DevTools → `Application` → `Local Storage`.
5. Изучить `sw.js` и проверить работу кэша.
6. Изучить `manifest.json` и вкладку `Manifest`.
7. Проверить установку PWA.
8. Выполнить доработки из `STUDENT_TASKS.md`.
9. Подготовить отчёт с пояснением, что уже было реализовано в шаблоне, а что вы добавили самостоятельно.
